## 摘要

现代大语言模型（LLM）在后训练阶段依赖强化学习来推进特定能力，但将多种能力整合到一个模型中仍然困难。现有方法，例如离策略微调（Off-Policy Finetune）和 Mix-RL，要么效率低下，要么损失性能。在本文中，我们提出多教师在策略蒸馏（Multi-teacher On-Policy Distillation, MOPD），这是一种用于组合多个领域 RL 教师能力的后训练范式：我们首先针对每个领域运行专门化 RL，以获得一组领域教师，然后在学生自身的采样轨迹上把这些教师蒸馏到学生中。这消除了暴露偏差，并提供了稠密优化信号。在 Qwen3-30B-A3B 上，MOPD 优于 Mix-RL、Cascade RL、Off-Policy Finetune 和 Param-Merge 基线，几乎继承了每个教师的全部能力。MOPD 还支持领域教师的并行、独立开发，去除了多领域后训练中常见的跨领域耦合。MOPD 已部署于工业规模前沿模型 MiMo-V2-Flash 的后训练中，展示了它在前沿规模 LLM 能力整合中的实践价值。

## 1 引言

强化学习（RL）已经成为提升大语言模型（LLM）能力的重要方法（Ouyang et al., 2022; Schulman et al., 2017; Shao et al., 2024）。不同任务领域依赖不同的 RL 流水线。数学推理现在使用可验证答案 RL 训练（Guo et al., 2025; Shao et al., 2024）；软件工程使用可执行沙箱中的智能体式 RL（Jain et al., 2025; Wei et al., 2025）；指令遵循和创意写作使用基于评分规约的 RL（Ouyang et al., 2022）；面向搜索的智能体使用 Web 环境 RL（Jin et al., 2025; Nakano et al., 2022）。每条流水线都能可靠地提升模型在目标领域上的能力。然而，我们最终想要的是一个在所有这些领域都表现良好的单一模型。但是，构建这样的模型仍然是现代 LLM 后训练中的开放问题。

针对这一能力整合问题，既有方法可分为四类（表 1）。（1）Mix-RL（Yang et al., 2025）将来自所有领域的提示汇聚为一个数据集并运行联合 RL，但跨领域训练信号会相互干扰，产生跷跷板效应（Standley et al., 2020），并使联合模型落后于各领域教师。（2）Cascade RL（Wang et al., 2026）按顺序训练各领域，因此早期阶段的能力可能随后续阶段推进而衰退，且较长的整体训练会放大稳定性风险。（3）Off-Policy Finetune（Liu et al., 2025）训练各领域 RL 教师，然后通过 SFT 在它们的采样轨迹上微调学生；这种离策略监督会引入典型的暴露偏差（Ranzato et al., 2016）。（4）Param-Merge（Ilharco et al., 2023; Wortsman et al., 2022）在权重空间中平均教师权重或组合任务向量，但融合后的模型通常不稳定，且无法匹配所有教师的完整能力。简言之，这些方法在学习效率、可达到峰值和训练稳定性之间相互折中，没有留下令人满意的中间方案。

**表 1：** 从实践中重要的三个维度比较能力整合范式。MOPD 是唯一同时实现稠密优化、在策略训练和可并行流水线的方法。

| 方法 | 稠密优化 | 在策略 | 可并行 |
| --- | --- | --- | --- |
| Param-Merge | × | - | ✓ |
| Off-Policy Finetune | ✓ | × | ✓ |
| Mix-RL | × | ✓ | × |
| Cascade RL | × | ✓ | × |
| MOPD（本文） | ✓ | ✓ | ✓ |

为克服这些限制，我们提出多教师在策略蒸馏（MOPD），这是一种直接在策略空间而非权重空间或静态数据集空间中进行多教师能力整合的后训练范式。MOPD 首先在每个领域独立执行专门化 RL，以获得一组强领域教师。随后，通过在策略蒸馏方法（Agarwal et al., 2024; Gu et al., 2026; Hinton et al., 2015）把这些教师的能力蒸馏到单个学生模型中。该设计具有三项结构性优点：（1）训练信号是稠密的，并且直接来自每个专门化教师，因此学生继承了每个教师的大部分能力；（2）训练完全在学生自身的采样轨迹分布上运行，从构造上消除了暴露偏差；（3）能力整合通过按提示路由在策略空间中实现，而不是通过权重空间融合实现，因此过程显著更稳定。

我们在两个不同规模和架构的基础模型上经验验证了 MOPD。在 Qwen3-30B-A3B（Yang et al., 2025）上，我们在三个领域（数学、指令遵循、软件工程（Jimenez et al., 2024; OpenAI, 2024））比较 Mix-RL、Cascade RL、Off-Policy Finetune、Param-Merge 和 MOPD；MOPD 在归一化分数上领先最强基线 5.5 个点（0.937 vs. 0.882）。我们进一步将 MOPD 部署到工业规模模型 MiMo-V2-Flash（Team et al., 2026）上，展示其在前沿规模设置中的实践有效性。进一步分析验证了 top-k 蒸馏实例化，确认同源教师对稳定优化至关重要，并展示了多轮迭代精炼的收益。

我们的贡献总结如下。（i）我们提出 MOPD，这是一种后训练范式，它通过在学生自身采样轨迹上提供的稠密、token 级监督来实现多教师能力整合。（ii）我们在 Qwen3-30B-A3B 上验证了 MOPD，表明它优于现有能力整合基线，并进一步将其应用于工业规模的 MiMo-V2-Flash 以确认实践可行性。

## 2 相关工作

**面向 LLM 的强化学习。** 在带可验证奖励的强化学习（RLVR）中，PPO 风格（Schulman et al., 2017）和 GRPO 风格（Shao et al., 2024）的结果奖励优化已经成为标准实践。这些 RL 流水线覆盖多样化的任务景观，包括数学推理（Guo et al., 2025; Shao et al., 2024）、软件工程（Jain et al., 2025; Wei et al., 2025）、指令遵循和创意写作（Ouyang et al., 2022）以及搜索智能体（Jin et al., 2025; Nakano et al., 2022）。每条这样的流水线都会产生强领域专家，而把它们整合到一个模型中自然引出了多领域/多任务问题。对于这一设置，先前工作主要采取两条路线：混合 RL（Yang et al., 2025），它把不同领域的样本放入同一个训练 batch（每个样本仍使用自己的领域奖励和优势）；以及级联 RL（Wang et al., 2026），它逐个训练各领域。

**面向 LLM 的蒸馏。** 经典蒸馏（Hinton et al., 2015; Kim and Rush, 2016）在固定的教师补全语料上最小化前向 KL，因此是离策略的，学生在推理时的轨迹可能偏离训练分布。在策略蒸馏通过在学生自身采样轨迹上训练，并把教师作为评分信号来解决这一问题，例如 MiniLLM（Gu et al., 2026）（反向 KL）以及把教师作为奖励的变体（Agarwal et al., 2024; Meng et al., 2024）。然而，它局限于单个教师和单个领域。MOPD 保留“学生采样、教师评分”的模板，并将其扩展到多个按提示路由的领域教师，且在生产规模基础模型上得到验证。

**模型合并。** 模型合并在权重空间中融合多个模型检查点，以在不增加额外训练的情况下获得组合模型。Model Soups（Wortsman et al., 2022）对来自相同初始化、独立微调的模型权重求平均。Task arithmetic（Ilharco et al., 2023）通过在权重空间中加减任务向量来组合能力，后续工作则通过 TIES-Merging（Yadav et al., 2023）、DARE（Yu et al., 2024）和 AdaMerging（Yang et al., 2024）来缓解参数冲突。

## 3 方法

### 3.1 流水线

MOPD 将后训练组织为三个顺序阶段：通用 SFT、领域专门化 RL 训练，以及最终的 MOPD 过程，该过程把各领域专家融合为单个模型（图 1）。

- **阶段 1：通用 SFT。** 我们首先在覆盖所有关注基础能力的广泛语料上微调基础模型，得到一个 SFT 检查点。该检查点作为阶段 2 中领域专门化 RL 训练的共享初始化。它也用作阶段 3 中 MOPD 学生的初始化。
- **阶段 2：领域专门化 RL 训练。** 在这一阶段，对于每个领域 \(d\)，我们从阶段 1 检查点出发训练一个领域专家 \(\pi_{\phi_d}\)。每个专家都使用最适合其领域的 RL 配方训练，例如数学使用可验证答案 RL，软件工程使用可执行沙箱智能体 RL。各领域训练彼此独立，可以完全并行执行。
- **阶段 3：MOPD。** 学生从阶段 1 的 SFT 检查点初始化，阶段 2 的领域专家 \(\{\pi_{\phi_d}\}\) 被冻结并构成教师组；随后我们在多领域数据集上训练学生，其中每个优化步骤执行以下操作：（1）从数据集中采样一批提示；（2）学生为每个提示生成一条轨迹，并记录轨迹上的逐 token 概率分布；（3）根据任务领域，将每条轨迹分发给对应领域教师，然后教师在该轨迹上预填充以获得逐 token 概率分布；（4）通过最小化学生与领域教师在该轨迹上的逐 token 反向 KL 来更新学生。训练后，阶段 3 得到一个获得了所有领域能力的统一模型。

**图 1：** 三阶段 MOPD 流水线概览。阶段 1 产生共享 SFT 检查点；阶段 2 并行训练各领域教师；阶段 3 反复执行：（i）从领域混合中采样提示，（ii）将其路由到匹配教师，（iii）让学生生成轨迹，（iv）让教师在轨迹上预填充以获得 token 级 log-probability，（v）使用教师信号更新学生。

相较于现有能力整合范式，MOPD 具有若干结构性优势：

- **无暴露偏差。** 学生在自身采样轨迹上训练，因此训练与推理的状态分布从构造上匹配。
- **稠密逐 token 监督。** 教师在轨迹的每个 token 处提供概率分布。这比标准 RL 中使用的轨迹级奖励稠密得多，降低方差并提高样本效率。
- **在策略空间中稳定整合。** 能力通过将每个提示路由到匹配教师来合并，而不是对教师参数求平均或做任务算术。这避免了权重空间融合中常见的不稳定性。
- **模块化且并行的教师。** 教师 RL 训练完全并行运行，且每个教师可以使用自己的超参数配置。某个教师 RL 训练中的意外失败或超参数调整不会影响其他教师的训练。
- **同源教师稳定性。** 由于每个教师都由领域特定 RL 从初始化学生的同一个 SFT 检查点产生，教师和学生共享高度对齐的策略分布。这确保了较低的初始 KL 散度，并在整个蒸馏过程中实现平滑、稳定的优化（§4.4.2）。

### 3.2 算法

在阶段 3 中，学生针对被分发教师的逐 token 反向 KL 进行优化：

$$
L_{\text{rev-KL}} =
\mathbb{E}_{x,y \sim \pi_\theta}
\left[
\frac{1}{|y|}
\sum_t \sum_v
\pi_\theta(v)
\log \frac{\pi_\theta(v)}{\pi_{\phi_d}(v)}
\right],
\tag{1}
$$

其中 \(\pi_{\phi_d}\) 是分发给提示 \(x\) 的教师，\(\pi_\theta(v)\)、\(\pi_{\phi_d}(v)\) 分别是 \(\pi_\theta(v \mid x, y_{<t})\)、\(\pi_{\phi_d}(v \mid x, y_{<t})\) 的简写（下文同样使用该简写）。接下来两个小节给出该目标的两种高效实现。

#### 3.2.1 策略梯度实现

沿用 MiniLLM（Gu et al., 2026），我们把 MOPD 蒸馏表述为一个 RL 过程。式（1）的梯度为

$$
\nabla_\theta L =
-\mathbb{E}_{x,y}
\left[
\frac{1}{|y|}
\sum_t
\log \frac{\pi_{\phi_d}(y_t)}{\pi_\theta(y_t)}
\nabla_\theta \log \pi_\theta(y_t)
\right],
\tag{2}
$$

这恰好是策略梯度形式，其中教师-学生 log 差异作为逐 token 优势：

$$
\hat{A}_{\text{MOPD},t}
= \operatorname{sg}
\left[
\log \pi_{\phi_d}(y_t) - \log \pi_\theta(y_t)
\right].
\tag{3}
$$

为保证训练稳定性，我们对优势做双侧裁剪：

$$
\hat{A}^{\text{clip}}_{\text{MOPD},t}
= \operatorname{clip}(\hat{A}_{\text{MOPD},t}, -A_{\max}, +A_{\max}),
$$

得到最终损失：

$$
L^{\text{PG}}_{\text{MOPD}}(\theta)
=
-\mathbb{E}_{x,y}
\left[
\frac{1}{|y|}
\sum_t
\hat{A}^{\text{clip}}_{\text{MOPD},t}
\log \pi_\theta(y_t)
\right].
\tag{4}
$$

这一形式可以直接放入现有 PPO/GRPO 训练框架；唯一变化是优势计算。

#### 3.2.2 Top-\(k\) 蒸馏实现

策略梯度形式在每个轨迹位置只使用单个采样 token。一个方差更低的替代方案是在教师的 top-\(k\) token 上蒸馏（Peng et al., 2024），它在保持实现简单的同时利用了更多教师分布。令

$$
T^d_t = \operatorname{TopK}_k(\pi_{\phi_d}(\cdot \mid x, y_{<t}))
$$

为位置 \(t\) 处教师下的 top-\(k\) token 集。我们使用如下蒸馏损失：

$$
L^{\text{TopK}}_{\text{MOPD}}(\theta)
=
\mathbb{E}_{x,y}
\left[
\frac{1}{|y|}
\sum_t
\sum_{v \in T^d_t}
\left(
\pi_\theta(v)\log \frac{\pi_\theta(v)}{\pi_{\phi_d}(v)}
- \pi_\theta(v) + \pi_{\phi_d}(v)
\right)
\right],
\tag{5}
$$

其中 \(\pi_\theta(v)\) 和 \(\pi_{\phi_d}(v)\) 都以 \((x, y_{<t})\) 为条件；为简洁起见，我们省略条件。相较于标准反向 KL，我们额外加入了 \(\pi_{\phi_d}(v)-\pi_\theta(v)\) 项。该额外项修正了 top-\(k\) 截断引入的偏差，确保当 top-\(k\) token 上 \(\pi_\theta=\pi_{\phi_d}\) 时损失达到最小；没有它，朴素 top-\(k\) 截断反向 KL 不具备这一性质。除正确性之外，top-\(k\) 形式还降低了基础设施负担：它使教师预填充载荷足够小，可以像轻量奖励信号一样传输；相比之下，全词表蒸馏要求为每个 token 传输整个词表分布。

### 3.3 基础设施

阶段 3 中新增的操作是教师预填充：每条学生采样轨迹都必须通过被分发教师，以获得逐 token log-probability（或 top-\(k\) logits）。一个自然基线是把教师预填充折叠进 RL 训练循环，但这会使基础设施复杂化，并增加额外的串行延迟。我们观察到，教师预填充计算与 RL 中的奖励计算性质相同，因此我们改为把每个领域教师部署为一个独立的预填充服务，位于 RL trainer 之外。

运行时流程如下。学生采样器持续生成轨迹；一旦某个序列的轨迹完成，trainer 就向对应教师服务发起异步预填充请求。教师服务计算并返回逐 token log-probability。由于教师预填充与其他序列的采样在时间上重叠，MOPD 的墙钟成本由采样本身主导，教师成本基本隐藏在学生采样之后。在我们的部署中，教师几乎没有贡献可测量的墙钟开销。

## 4 实验

我们在受控条件下（§4.1-4.2）于 Qwen3-30B-A3B 上比较 MOPD 与四种能力整合范式，并在工业规模前沿模型 MiMo-V2-Flash 上验证它（§4.3）。最后，§4.4 给出分析。

### 4.1 实验设置

**SFT 模型。** 我们的起始检查点是 Qwen3-30B-A3B-Base（Yang et al., 2025）。为使其成为 RL 的强起点，我们在覆盖所有目标领域的广覆盖语料上微调它。所有基线和 MOPD 都从同一个 SFT 检查点开始。

**任务与评估。** 我们的任务跨越三个领域：数学（在 AIME25 和 AIME26 上评估）、指令遵循（在 IFBench（Pyatkin et al., 2025）和 IFEval（Zhou et al., 2023）上评估）以及软件工程（在 SWE-bench Verified（Jimenez et al., 2024）上评估）。训练数据来源和超参数见附录 A。

**基线。** 我们比较五种能力整合范式：（i）Mix-RL，在汇聚所有领域提示的数据集上做联合 RL；（ii）Cascade RL，按顺序应用各领域 RL，每个阶段从前一阶段初始化；（iii）Off-Policy Finetune，在阶段 2 教师产生的采样轨迹上对学生进行离线 SFT；（iv）Param-Merge，对阶段 2 教师做线性/任务算术合并；（v）MOPD（本文）。

**评估指标。** 三个领域在绝对提升空间上差异很大，因此原始准确率平均会过度加权提升空间更大的领域。为把各领域放到共同尺度上，我们报告归一化分数。令 \(D\) 表示评估领域集合。对每个 \(d \in D\)，令 \(s^s_d\) 为阶段 1 SFT 学生在该领域上的准确率，\(s^t_d\) 为对应阶段 2 领域专家准确率；对于某个领域准确率为 \(s_d\) 的方法，逐领域归一化分数为

$$
\tilde{s}_d = (s_d - s^s_d)/(s^t_d - s^s_d),
$$

它在阶段 1 学生处等于 0，在领域专家教师处等于 1。我们报告的主指标是在各领域上的均匀平均：

$$
\bar{\tilde{s}} = \frac{1}{|D|}\sum_{d\in D}\tilde{s}_d.
$$

大于 1 的值表示超过专家教师的提升，小于 0 的值表示相较阶段 1 学生发生退化。

#### 4.4.1 策略梯度与 top-\(k\) 蒸馏表现相当

表 4 的前两行报告了比较结果。Top-\(k\) 蒸馏在 Math 上表现相当，在 IF 和 SWE 上略差；两种形式达到总体相近的能力，归一化分数分别为 0.909 和 0.937。图 3 证实了这一点：在同源教师下，两种损失形式产生几乎重合的训练轨迹：数学准确率平滑上升，逐 token 学生-教师反向 KL 从已经较低的初始值（约 0.04）单调下降，策略熵在整个过程中稳定在约 0.30。当教师和学生共享紧密相关的分布时，学生采样轨迹集中于教师的高概率区域，因此两种梯度估计器获得相似的信息性信号并收敛到相近终点。这表明，在同源教师下，策略梯度形式已经足够稳定；引入 top-\(k\) 并不会带来额外稳定性或方差降低。

#### 4.4.2 同源教师稳定蒸馏

MOPD 的领域教师由学生经 RL 训练得到。这意味着教师和学生共享相似的策略分布。近期研究（Ko et al., 2026; Li et al., 2026）讨论了教师-学生策略散度与在策略蒸馏稳定性之间的关系。一个自然问题是：如果用一个更强但分布不同的外部模型替换教师，是否会得到更好结果？为测试这一点，我们设计了一个受控实验：在 MOPD 的 Math 领域中，我们用 Qwen3-235B-A22B 替换原始 Qwen3-30B-A3B Math RL 教师，前者是一个数学能力更强、规模显著更大的模型。所有其他设置（IF 和 SWE 教师、数据、超参数）保持不变。

表 4 的后两行报告了结果。尽管 Qwen3-235B-A22B 的绝对数学能力显著更强，MOPD 学生在两种损失变体下的数学性能都下降。图 3 揭示了机制。初始逐 token KL 比同源设置约高 5 倍（约 0.19 vs. 约 0.04），定量确认了巨大的分布差距。在策略梯度形式下，数学准确率逐渐下降，同时熵从 0.30 收缩到 0.21；当学生主要从教师低概率区域接收惩罚性梯度信号时，其策略收窄到单一模式。在 top-\(k\) 形式下，崩溃甚至更严重：训练在约第 18 步灾难性发散，KL 和熵都出现剧烈振荡，表明优化稳定性完全丧失。这些结果强调，MOPD 使用同源教师至关重要：紧密的分布对齐确保了整个训练过程中的稳定优化，带来可靠且一致的收益。

#### 4.4.3 多轮学生-教师演化

## 5 讨论

除经验收益之外，MOPD 在结构上解耦了多领域后训练内在的串行依赖，将能力生产（阶段 2 的各领域 RL）与能力整合（阶段 3 的蒸馏）分离。由此产生三项工作流结果：

- **并行开发。** 阶段 2 教师彼此独立：每个领域团队可以并发迭代自己的奖励、沙箱和数据流水线，没有固定顺序。在算力充足时，这会直接提高团队整体开发吞吐。
- **配方级解耦。** 每个领域都可以自由选择自己的 RL 配方（算法、采样轨迹过程、奖励函数、超参数等），无需担心与其他领域冲突，也无需担心自己的优化选择干扰其他领域。
- **风险隔离。** RL 训练通常需要反复调试算法和超参数（无论是为了进一步提升能力，还是仅仅为了解决稳定性问题），而每次这种调整都会迫使训练重启。在联合多领域 RL 下，重启会让整个运行回到起点；在 MOPD 的并行教师开发下，重启被限制在单个受影响领域中，其他教师可以继续不受干扰。

## 6 结论

我们提出了 MOPD（Multi-Teacher On-Policy Distillation，多教师在策略蒸馏），这是一种在策略空间中执行多领域能力整合的后训练范式：学生从自身采样轨迹中采样，每个提示被路由到对应冻结领域教师，而教师在每个位置提供稠密、token 级 log-probability 信号。在 Qwen3-30B-A3B 上，MOPD 在 Mix-RL、Cascade RL、Off-Policy Finetune 和 Param-Merge 中取得最佳整体整合效果，领先次优整合方法 5.5 个归一化分数点，同时在每个领域关闭 91-95% 的学生-教师提升空间；在 MiMo-V2-Flash 上，同一配方在多数 benchmark 上匹配或超过对应教师。我们的分析进一步表明，策略梯度和 top-\(k\) 损失形式表现相当，同源教师对稳定优化至关重要，且该流水线支持多轮学生-教师演化。结合 §5 中讨论的结构性解耦，我们认为 MOPD 为规模化 LLM 后训练提供了一条工程路径，支持并行开发和快速整合。

## 附录 A 训练细节

**训练数据。** 我们的训练任务覆盖三个领域：

1. **数学。** SFT 数据主要来自 Mixture-of-Thoughts 数据集的数学子集（Hugging Face, 2025）；RL 训练数据从许多开源数据集中收集并过滤，包括 BigMath（Albalak et al., 2025）和 ORZ（Hu et al., 2025）。最大序列长度为 32,768。
2. **指令遵循。** SFT 提示按照 IFBench（Pyatkin et al., 2025）介绍的方法构造，并在 gpt-oss-120b（OpenAI et al., 2025）上蒸馏；RL 训练数据同样按照 IFBench 配方合成。最大序列长度为 32,768。
3. **软件工程。** SFT 数据通过开源模型从 R2E-Gym 任务中蒸馏得到；RL 训练数据使用 R2E-Gym-Lite（Jain et al., 2025）。最大序列长度为 65,536，交互轮数上限为 50。

**超参数。** 对于 RL 训练，我们使用带动态采样（Dynamic Sampling）（Yu et al., 2025）的在策略 GRPO（Shao et al., 2024）算法：每次采样轨迹生成的数据只用于一次梯度更新，然后丢弃。对于各领域 RL，我们将学习率设为 \(3 \times 10^{-6}\)。对于 Math 和 IF 训练，我们将 batch size（BS）设为 144，每个提示 \(N=8\) 条采样轨迹，训练约 175K 个序列。对于 SWE，我们将 BS 设为 80，\(N=8\)，训练约 150K 个序列。Cascade RL 使用与各领域 RL 相同的配置，并按 IF→Math→SWE 顺序训练各领域。对于 Mix-RL，我们将学习率设为 \(4 \times 10^{-6}\)，BS 设为 256，\(N=8\)，每个 batch 的领域比例为 Math : IF : SWE = 0.35 : 0.35 : 0.3。对于 MOPD，我们不使用动态采样，并将 BS 设为 2048，\(N=1\)，每个 batch 的领域比例为 Math : IF : SWE = 0.35 : 0.35 : 0.3。策略梯度形式默认的优势裁剪最大值为 5；top-\(k\) 蒸馏形式默认 \(k=64\)。

## 附录 B 评估细节

我们描述所有报告结果使用的评估协议。对于两个数学 benchmark AIME25 和 AIME26，我们对每个问题采样 32 次，并报告平均准确率（avg@32），以降低低样本数、高难度数学评估固有的方差。对于指令遵循，我们在 IFBench 和 IFEval 上对每个问题评估一次。对于软件工程，我们在 SWE-bench Verified 上对每个任务评估一次。

对于所有 benchmark，解码使用采样温度 1.0，不应用 top-\(p\) 或 top-\(k\) 截断。
